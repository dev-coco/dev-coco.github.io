数据类型

  1. 文本型

    2. 一般指文字或者特殊符号，文本不能用于计算。

  2. 数值型

    3. 单元格格式

    1. 数字
      • 123
      • 3.14159
      • 1,000
    2. 货币
      • ¥56
      • £30.00
    3. 日期
      • 2021/12/25
      • 5/2
      • 2022年3月4日星期五
    4. 时间
      • 10:12
      • 下午03:12:00
      • 2022-2-21 下午03:12:00
    5. 百分比
      • 130%
    6. 其它
      • (123) 456-7890
  3. 布尔型

    4. 一般用在逻辑判断，判断条件是否成立。布尔值只有两种值，TRUE 和 FALSE，TRUE 等于 1，FALSE 等于 0。

    所有的数据类型都可以进行比较，在不同类型的情况下：布尔型 > 文本型 > 数值型。

单元格条件

设置单元格格式不会影响数据类型。如果单元格的内容是数值，把单元格的格式改成文本格式，并不会改变数据的类型，用函数计算的时候也不会影响到结果。但是在某些情况下，修改单元格的格式会影响数据显示的样式，所以不要被单元格显示结果欺骗了。下面看一个例子。

使用 SUM 函数计算 A2 和 A3 的结果，正常情况下 A2 加 A3 的结果应该是 2，但是却返回 3。 因为 A2 和 A3 设置了正整数条件格式，小数点会按照四舍五入计算。在函数栏可以看到 A2 单元格真实的值，A2 实际是 1.4，A3 是 1.1。

用 SUM 函数计算的时候等于 1.5，因为 SUM 函数返回的结果是正整数，小数点后面的值会按照四舍五入计算，所以就会返回 3。

数据类型默认排版

在默认情况下，文本型靠左，数值型靠右，布尔值居中。
